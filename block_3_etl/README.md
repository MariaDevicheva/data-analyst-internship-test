# Music Analytics ETL Pipeline

**Python | Pandas | SQL | SQLite | Docker | ETL**

## Описание проекта

Проект представляет собой ETL-конвейер для извлечения, очистки, преобразования, загрузки и последующего анализа данных из нескольких разнородных источников.

В рамках проекта реализована обработка данных о клиентах, заказах, товарах, платежах и пользовательских событиях с последующей загрузкой в аналитическую базу данных SQLite и выполнением аналитических SQL-запросов.

---

## Цель проекта

Основная цель проекта — продемонстрировать навыки:

* работы с различными форматами данных;
* разработки ETL-процессов на Python;
* очистки и валидации данных;
* контроля качества данных;
* проектирования структуры базы данных;
* выполнения аналитических SQL-запросов;
* контейнеризации приложения с использованием Docker.

## DDL-схема базы данных

В папке `ddl/` представлены SQL-скрипты с описанием структуры аналитических таблиц SQLite.

В текущей реализации таблицы создаются автоматически при загрузке очищенных DataFrame в SQLite через `pandas.to_sql()`.

DDL-файлы используются как документация схемы базы данных и позволяют явно зафиксировать структуру таблиц, формируемых в процессе ETL.

Основной файл:

```text
ddl/schema.sql
```

## ETL-процесс

### Extract

Извлечение данных реализовано из нескольких источников:

* CSV;
* JSON;
* XLSX;
* XML.

### Transform

На этапе преобразования выполняется:

* удаление дубликатов;
* обработка пропусков;
* приведение типов данных;
* проверка корректности дат;
* проверка числовых значений;
* очистка некорректных записей;
* расчёт поля `order_amount`.

Примеры выявленных проблем качества данных:

* некорректные даты;
* ошибочные значения сумм;
* пропущенные значения в обязательных полях;
* дублирующиеся записи.

### Load

После очистки данные загружаются в SQLite-базу данных:

* customers;
* orders;
* products;
* payments;
* events.

## Локальный запуск

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск ETL-конвейера

```bash
python src/main.py
```

### Запуск аналитики

```bash
python run_analytics.py
```

После выполнения формируются CSV-файлы с результатами анализа в папке:

```text
results/
```

## Аналитические запросы

Реализованы следующие аналитические отчёты:

1. Анализ заказов по статусам.
2. Анализ товарных категорий.
3. Анализ способов оплаты.
4. Анализ пользовательской активности.
5. Анализ наиболее ценных клиентов.

Результаты сохраняются в папке:

```text
results/
```

## Результат

В рамках проекта реализован полный ETL-конвейер, включающий извлечение данных из различных источников, очистку и контроль качества данных, загрузку в аналитическую базу данных SQLite и выполнение аналитических SQL-запросов.

Проект демонстрирует практические навыки разработки ETL-процессов, подготовки данных, работы с SQL, контроля качества данных и построения воспроизводимых аналитических решений.
