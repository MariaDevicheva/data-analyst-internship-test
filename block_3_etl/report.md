# Аналитический отчёт по ETL-конвейеру и обработке данных

## Описание проекта

В рамках проекта был разработан ETL-конвейер для извлечения, очистки, преобразования и загрузки данных из нескольких источников в единую аналитическую базу данных SQLite.

После выполнения ETL-процесса был проведён анализ подготовленных данных с целью оценки пользовательской активности, структуры заказов, способов оплаты и характеристик клиентской базы.

---

## Объём обработанных данных

После завершения ETL-конвейера были сформированы следующие аналитические таблицы:

| Таблица   | Количество записей |
| --------- | -----------------: |
| customers |                684 |
| orders    |               1113 |
| products  |                581 |
| payments  |                695 |
| events    |               1450 |

В процессе обработки были устранены дубликаты, пропущенные значения и некорректные записи, что позволило повысить качество данных перед загрузкой в аналитическую базу.

# 1. Анализ заказов по статусам

## Результаты

| Статус     | Количество заказов | Сумма заказов |
| ---------- | -----------------: | ------------: |
| processing |                378 |       303 294 |
| completed  |                393 |       301 296 |
| cancelled  |                342 |       252 233 |

## Выводы

Наибольший объём выручки приходится на заказы со статусом **processing** и **completed**.

Количество завершённых заказов сопоставимо с количеством заказов, находящихся в обработке, что может свидетельствовать о стабильном потоке клиентских операций.

При этом доля отменённых заказов остаётся заметной и требует дополнительного анализа причин отмен.

## Рекомендации

* проанализировать причины отмены заказов;
* контролировать скорость обработки заказов;
* отслеживать конверсию заказов из статуса processing в completed.

# 3. Анализ способов оплаты

## Результаты

Используются три основных способа оплаты:

* card;
* paypal;
* bank_transfer.

Наибольшее количество операций приходится на оплату банковской картой.

Средний размер платежа находится в диапазоне от 1390 до 1540 денежных единиц в зависимости от способа оплаты.

## Выводы

Банковская карта остаётся наиболее востребованным способом оплаты среди пользователей.

При этом средний чек по всем способам оплаты остаётся сопоставимым, что свидетельствует об отсутствии существенных различий в поведении пользователей в зависимости от выбранного платёжного инструмента.

## Рекомендации

* поддерживать удобство карточных платежей;
* контролировать стабильность платёжных сервисов;
* анализировать влияние способа оплаты на конверсию заказов.

# 5. Наиболее ценные клиенты

## Результаты

Были определены клиенты с наибольшим суммарным объёмом заказов.

Максимальный объём покупок одного клиента составил более 6 000 денежных единиц.

Топ-10 клиентов обеспечивают значительную долю оборота платформы.

## Выводы

В клиентской базе присутствует группа пользователей с высокой покупательской активностью.

Подобные клиенты являются наиболее ценным сегментом аудитории и требуют отдельного внимания со стороны бизнеса.

## Рекомендации

* использовать программы лояльности;
* развивать персонализированные предложения;
* внедрять механизмы удержания наиболее активных клиентов.

# Итоговые выводы

В рамках проекта был реализован полный цикл ETL-обработки данных: от извлечения данных из различных источников до формирования аналитической базы данных и выполнения последующего анализа.

Проведённое исследование позволило выявить структуру заказов, определить наиболее прибыльные товарные категории, оценить предпочтения пользователей при выборе способов оплаты и исследовать особенности пользовательской активности.

Полученные результаты могут использоваться для принятия продуктовых решений, повышения эффективности маркетинговых активностей, развития клиентской базы и дальнейшего совершенствования аналитической инфраструктуры компании.
