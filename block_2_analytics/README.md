# Задача 2. Анализ цепочек идентификаторов

## Описание

Рассмотрена задача объединения связанных идентификаторов пользователей.

Даны таблицы:

```text
users(id, new_id)
links(id1, id2)
```

Таблица `links` содержит историю изменений идентификаторов.

Пример цепочки:

```text
1 → 2 → 3 → 5
```

Все идентификаторы внутри одной цепочки должны быть сопоставлены единому значению.

---

## Реализация

Решение включает два подхода:

### Рекурсивный

Использование:

```sql
WITH RECURSIVE
```

для обхода цепочек произвольной длины.

### Нерекурсивный

Использование серии:

```sql
LEFT JOIN
```

для последовательного раскрытия связей.

Файл решения:

```text
sql_python_tasks_2_4/task_2_id_chains.sql
```

Результат выполнения:

```text
results/task_2_result.csv
```

# Задача 4. Поиск симметричных пар

## Описание

Необходимо найти записи с одинаковыми значениями:

* name;
* category.

При этом пары:

```text
(id1, id2)
(id2, id1)
```

считаются одинаковыми.

---

## Реализация

Использован self join таблицы и условие:

```sql
i1.id < i2.id
```

что позволяет исключить зеркальные дубликаты.

Файл решения:

```text
sql_python_tasks_2_4/task_4_symmetric_pairs.sql
```

Результат:

```text
results/task_4_result.csv
```
