# Аналитический отчет по исследованию и прогнозированию пользовательской активности

## Описание проекта

В рамках проекта выполнен комплексный анализ временного ряда пользовательской активности с последующим построением прогнозной модели и решением прикладных задач по обработке данных средствами SQL и Python.

Цель исследования заключалась в выявлении закономерностей поведения пользователей, оценке динамики ключевого показателя во времени и построении прогноза на будущие периоды для поддержки принятия решений на основе данных.

Дополнительно были рассмотрены задачи по дедупликации данных, нормализации текстовых значений и поиску связей между объектами с использованием SQL.

---

## Бизнес-контекст

Во многих цифровых продуктах и онлайн-сервисах анализ пользовательской активности является одним из ключевых инструментов оценки состояния бизнеса.

Понимание долгосрочных трендов, сезонных колебаний и прогнозируемой динамики позволяет:

* оценивать устойчивость пользовательской базы;
* планировать маркетинговые активности;
* прогнозировать нагрузку на инфраструктуру;
* своевременно выявлять снижение вовлеченности аудитории;
* принимать решения на основе прогнозируемых показателей.

В рамках данного исследования в качестве целевой метрики использовался показатель количества пользователей сервиса по месяцам.

---

# Задача 1. Анализ данных и прогнозирование

## Исследование данных

В качестве исходных данных использовался временной ряд пользовательской активности, содержащий исторические наблюдения по месяцам.

На этапе разведочного анализа данных были выполнены:

* исследование структуры датасета;
* проверка качества данных;
* анализ распределения целевой переменной;
* визуализация динамики показателя во времени;
* выявление трендов и сезонных компонентов.

### Основные наблюдения

Анализ временного ряда показал наличие нескольких характерных особенностей.

#### Долгосрочный тренд

На протяжении рассматриваемого периода наблюдается постепенное снижение значения исследуемого показателя.

Подобная динамика может быть связана с:

* изменением поведения пользователей;
* усилением конкурентной среды;
* естественным снижением активности аудитории;
* особенностями формирования исторических данных.

#### Сезонность

Во временном ряду присутствуют повторяющиеся циклические колебания, указывающие на наличие сезонного эффекта.

Сезонные закономерности являются важным фактором при построении прогностической модели и были учтены при формировании признакового пространства.

---

## Построение прогнозной модели

Для решения задачи прогнозирования использовался алгоритм машинного обучения:

RandomForestRegressor

В модель были включены следующие признаки:

* временной индекс наблюдения;
* номер месяца;
* сезонные компоненты:

  * month_sin;
  * month_cos;
* дополнительные признаки, содержащиеся в исходном наборе данных.

Использование сезонных признаков позволило учитывать цикличность временного ряда и повысить качество прогнозирования.

---

## Оценка качества модели

Для оценки точности прогнозирования использовались стандартные метрики регрессионного анализа.

### MAE

653.72

Средняя абсолютная ошибка модели составляет около 654 единиц целевого показателя.

### MAPE

7.87%

Средняя относительная ошибка прогнозирования не превышает 8%.

Полученные значения свидетельствуют о хорошем качестве модели для решения задачи среднесрочного прогнозирования пользовательской активности.

---

## Интерпретация результатов прогнозирования

Построенная модель показывает сохранение пользовательской активности на относительно стабильном уровне в прогнозируемом периоде.

Ожидаемый диапазон значений составляет примерно:

9400–10100 пользователей

Существенного роста аудитории модель не прогнозирует. При сохранении текущих тенденций наиболее вероятным сценарием является умеренное снижение активности с сохранением сезонных колебаний.

---

# Задача 2. Анализ цепочек идентификаторов пользователей

## Описание задачи

В информационных системах нередко возникают ситуации, когда одному пользователю на протяжении жизненного цикла соответствуют различные идентификаторы.

Для корректного анализа данных требуется объединить такие записи и определить единый идентификатор пользователя.

В рамках задачи был реализован алгоритм поиска связанных идентификаторов на основе рекурсивного обхода графа связей.

---

## Результат

В результате обработки были успешно выявлены группы связанных идентификаторов и выполнена их консолидация в единые сущности.

Решение реализовано средствами SQL с использованием рекурсивных CTE.

---

# Задача 3. Очистка и дедупликация текстовых данных

## Описание задачи

При анализе пользовательских данных часто встречаются текстовые значения, содержащие визуально одинаковые символы из различных алфавитов.

Например:

* Александр
* Aлександр

Несмотря на визуальное сходство, подобные записи считаются различными строками и приводят к появлению скрытых дубликатов.

---

## Результат

Был реализован алгоритм нормализации текстовых значений, включающий:

* замену визуально схожих символов;
* приведение строк к единому регистру;
* группировку логических дубликатов;
* присвоение единого идентификатора группе записей.

Результатом стала корректная дедупликация пользовательских данных.

---

# Задача 4. Поиск дублирующихся объектов

## Описание задачи

Для повышения качества данных была решена задача поиска объектов с одинаковыми характеристиками внутри одной категории.

Решение реализовано с использованием self join и механизма исключения зеркальных дубликатов.

---

## Результат

Получен уникальный набор совпадающих объектов без повторяющихся комбинаций.

Подобный подход может использоваться для контроля качества справочников, каталогов товаров и клиентских данных.
