# SQL-анализ музыкального стримингового сервиса

## Описание проекта

В рамках данного проекта выполнен аналитический SQL-анализ данных музыкального стримингового сервиса.

Целью работы является исследование пользовательской активности, популярности музыкального контента и особенностей потребления музыки с использованием реляционной модели данных и SQL-запросов различного уровня сложности.

В ходе решения использовались:

* объединение данных из нескольких таблиц (JOIN);
* агрегатные функции;
* группировки (GROUP BY);
* оконные функции;
* Common Table Expressions (CTE);
* сортировка и ранжирование данных.

---

## Структура данных

База данных содержит информацию об исполнителях, альбомах, песнях, жанрах и истории прослушиваний пользователей.

### artists

Справочник исполнителей.

| Поле      | Описание                             |
| --------- | ------------------------------------ |
| artist_id | Уникальный идентификатор исполнителя |
| name      | Имя исполнителя                      |
| country   | Страна исполнителя                   |

---

### albums

Информация об альбомах.

| Поле         | Описание                            |
| ------------ | ----------------------------------- |
| album_id     | Уникальный идентификатор альбома    |
| title        | Название альбома                    |
| artist_id    | Идентификатор основного исполнителя |
| release_year | Год выпуска                         |

Связь:

* один исполнитель может иметь несколько альбомов;
* каждый альбом принадлежит одному основному исполнителю.

---

### songs

Информация о музыкальных композициях.

| Поле         | Описание                       |
| ------------ | ------------------------------ |
| song_id      | Уникальный идентификатор песни |
| title        | Название песни                 |
| album_id     | Идентификатор альбома          |
| release_year | Год выпуска                    |

Связь:

* один альбом содержит множество песен;
* каждая песня принадлежит одному альбому.

---

### song_artists

Таблица связи песен и исполнителей.

Используется для хранения коллабораций и совместных треков.

| Поле      | Описание                  |
| --------- | ------------------------- |
| song_id   | Идентификатор песни       |
| artist_id | Идентификатор исполнителя |

Особенности:

* одна песня может исполняться несколькими артистами;
* один артист может участвовать в большом количестве песен;
* реализует связь «многие ко многим».

---

### genres

Справочник музыкальных жанров.

| Поле     | Описание                       |
| -------- | ------------------------------ |
| genre_id | Уникальный идентификатор жанра |
| name     | Название жанра                 |

---

### song_genres

Таблица связи песен и жанров.

| Поле     | Описание            |
| -------- | ------------------- |
| song_id  | Идентификатор песни |
| genre_id | Идентификатор жанра |

Особенности:

* одна песня может относиться к нескольким жанрам;
* один жанр может включать множество песен;
* реализует связь «многие ко многим».

---

### listening_logs

Журнал пользовательских прослушиваний.

Каждая запись соответствует факту прослушивания определённой песни пользователем.

| Поле        | Описание                   |
| ----------- | -------------------------- |
| user_id     | Идентификатор пользователя |
| song_id     | Идентификатор песни        |
| listen_time | Дата и время прослушивания |
| device      | Тип устройства             |
| region      | Регион пользователя        |

Используется для:

* анализа популярности песен;
* оценки активности пользователей;
* построения временных рядов прослушиваний;
* исследования региональных музыкальных предпочтений.

---

## Выполненные задачи

### 1. Самый популярный Rock-альбом

Определение альбома жанра Rock с максимальным количеством прослушиваний.

Результат позволяет выявить наиболее востребованный музыкальный релиз среди пользователей сервиса.

---

### 2. Исполнитель с наибольшим количеством песен в топ-20%

Для решения использовалась оконная функция `NTILE()`.

Задача позволяет определить исполнителя, чьи композиции наиболее часто попадают в группу самых популярных треков.

---

### 3. Альбом с максимальным количеством коллабораций

Анализ совместных композиций нескольких исполнителей на основе таблицы `song_artists`.

Результат позволяет оценить влияние коллабораций на музыкальный контент.

---

### 4. Динамика прослушиваний по месяцам

Построение временного ряда активности пользователей.

Используется для анализа сезонности и изменения пользовательского интереса во времени.

---

### 5. Популярность жанров по регионам

Исследование музыкальных предпочтений пользователей в различных регионах.

Результаты могут использоваться для персонализации рекомендаций и регионального маркетинга.
